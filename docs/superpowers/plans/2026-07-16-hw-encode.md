# Hardware Video Encoding Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add NVENC + VAAPI H.264 encoders behind the existing `Encoder` interface, chosen at runtime (NVENC → VAAPI → software) with a `--encoder` override and a clean software fallback — offloading encode from the CPU with no client change.

**Architecture:** A shared `Nv12Converter` (BGRA→NV12) reused by all encoders; new `NvencEncoder`/`VaapiEncoder` mirroring `SoftwareEncoder`; an `AutoEncoder` wrapper whose `open()` tries candidate factories in order; a `make_encoder(pref)` factory. `stream_main` builds the encoder via the factory. **Host-only — no protocol/client/Android change; stream stays plain H.264 with in-band SPS/PPS.**

**Tech Stack:** C++ + FFmpeg (libavcodec/libavutil/libswscale), CMake, GoogleTest.

## Global Constraints

- **Host-only.** No `client/`, `android/`, or protocol/HELLO change.
- **Interface unchanged:** every encoder implements `Encoder` (`open`/`extradata`/`encode`/`flush`) from `host/src/encoder.h`. `StreamDaemon` still takes `Encoder&`.
- **In-band SPS/PPS:** do NOT set `AV_CODEC_FLAG_GLOBAL_HEADER` on any encoder (headers ride in-band per IDR, as x264 `repeat-headers` does today) — the client decodes from in-band IDR headers, unchanged.
- **Selection:** `make_encoder(pref)` order — `auto`→NVENC,VAAPI,Software; a forced backend → only that one (its `open()` failure is fatal, so misconfig is visible). First candidate whose `open()` succeeds wins.
- **Low-latency CBR** matching `SoftwareEncoder`: `bit_rate`/`rc_max_rate` = bitrate, `rc_buffer_size` ~2 frames, `max_b_frames=0`, `gop_size=fps*2`.
- **New sources** join `droppix_core` in `host/CMakeLists.txt` (next to `src/software_encoder.cpp`, line ~26); new tests join `droppix_tests` (line ~111). FFmpeg is already linked (`PkgConfig::FFMPEG` = libavcodec libavutil libswscale) — hwcontext is in libavutil, no new deps.
- **Build/test env** (repo on CIFS no-exec mount): `distrobox enter droppix-dev -- bash -lc 'cmake -S host -B ~/droppix-build -DDROPPIX_BUILD_TESTS=ON && cmake --build ~/droppix-build -j && ctest --test-dir ~/droppix-build <-R filter> --output-on-failure'`. (GPU encode is NOT run in the container — only CPU-side pieces are unit-tested; hw encode is on-device.)
- Work on branch `feat/hw-encode` (off `master`). Commit after each task.

- [ ] **Step 1: Read** `software_encoder.cpp` + FFmpeg hwcontext (`libavutil/hwcontext.h`). VAAPI needs a device ctx + a frames ctx (NV12 sw_format), and per frame a CPU NV12 (via `Nv12Converter`) uploaded to a VAAPI surface.

- [ ] **Step 2: Implement.** Members: `AVCodecContext* ctx_; AVBufferRef* hw_device_ = nullptr; AVBufferRef* hw_frames_ = nullptr; AVPacket* pkt_; Nv12Converter conv_; AVFrame* vaapi_frame_ = nullptr; int width_,height_,fps_; int64_t frame_index_; std::map<int64_t,int64_t> pts_map_;`.
  - `open(w,h,fps,br)`: **device probe** — for `node` in `{nullptr, "/dev/dri/renderD128", "/dev/dri/renderD129", ..., "/dev/dri/renderD135"}`, attempt a full open on that node and keep the first that fully succeeds:
    ```cpp
    for (const char* node : nodes) {
      if (av_hwdevice_ctx_create(&hw_device_, AV_HWDEVICE_TYPE_VAAPI, node, nullptr, 0) < 0) { hw_device_ = nullptr; continue; }
      // frames ctx: VAAPI surfaces, sw NV12
      hw_frames_ = av_hwframe_ctx_alloc(hw_device_);
      auto* fc = (AVHWFramesContext*)hw_frames_->data;
      fc->format = AV_PIX_FMT_VAAPI; fc->sw_format = AV_PIX_FMT_NV12; fc->width = w; fc->height = h;
      if (av_hwframe_ctx_init(hw_frames_) < 0) { free_hw(); continue; }
      const AVCodec* codec = avcodec_find_encoder_by_name("h264_vaapi");
      ctx_ = avcodec_alloc_context3(codec);
      ctx_->width=w; ctx_->height=h; ctx_->pix_fmt=AV_PIX_FMT_VAAPI; ctx_->time_base={1,fps};
      ctx_->framerate={fps,1}; ctx_->gop_size=fps*2; ctx_->max_b_frames=0;
      ctx_->bit_rate=int64_t(br)*1000; ctx_->rc_max_rate=ctx_->bit_rate;
      ctx_->rc_buffer_size=int(ctx_->bit_rate/std::max(1,fps)*2);
      ctx_->hw_frames_ctx = av_buffer_ref(hw_frames_);
      av_opt_set(ctx_->priv_data, "rc_mode", "CBR", 0);
      // NO GLOBAL_HEADER (in-band SPS/PPS).
      if (avcodec_open2(ctx_, codec, nullptr) == 0) { /* success on this node */ break; }
      // failed on this node (e.g. NVDEC decode-only default) -> free + try next
      free_this_attempt();  // avcodec_free_context(&ctx_); unref hw_frames_/hw_device_; null them
    }
    if (!ctx_ || !avcodec_is_open(ctx_)) return false;
    if (!conv_.open(w,h)) return false;
    vaapi_frame_ = av_frame_alloc();               // reused per-encode target surface
    pkt_ = av_packet_alloc();
    // log which node won
    return pkt_ != nullptr;
    ```
    (Provide a small `free_this_attempt()` lambda/helper that frees `ctx_`, `hw_frames_`, `hw_device_` and nulls them so the next node starts clean.)
  - `encode()`: `AVFrame* nv12 = conv_.convert(frame);` then upload to a VAAPI surface and send:
    ```cpp
    av_frame_unref(vaapi_frame_);
    if (av_hwframe_get_buffer(hw_frames_, vaapi_frame_, 0) < 0) return {};
    if (av_hwframe_transfer_data(vaapi_frame_, nv12, 0) < 0) return {};
    int64_t idx = frame_index_++; vaapi_frame_->pts = idx; pts_map_[idx] = pts_us;
    if (avcodec_send_frame(ctx_, vaapi_frame_) < 0) return {};
    return drain();
    ```
  - `drain()`/`extradata()`/`flush()` mirror `SoftwareEncoder`. dtor: free `pkt_`, `vaapi_frame_`, `ctx_`, then `av_buffer_unref(&hw_frames_)`, `av_buffer_unref(&hw_device_)` (`conv_` frees itself). A failed `open()` (no encode-capable node) leaves everything null → safe destruct and the factory falls through.
